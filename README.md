1. Java 관련 명령어 (WAR 생성 등)
작업하신 결과물을 배포용 파일로 만들 때 필수적인 명령어입니다.

jar -cvf [파일명.war] . : 현재 폴더(.)의 모든 내용을 지정한 이름의 WAR 파일로 압축합니다.

jar -xvf [파일명.war] : 생성된 WAR 파일의 압축을 풉니다. (내용물 확인용)

java -version : 현재 설치된 Java 버전을 확인합니다.

javac -version : 컴파일러 버전을 확인합니다.



2. 파일 및 폴더 관리 (백업 시 유용)
일자별로 폴더를 관리하실 때 마우스보다 훨씬 빠르게 작업할 수 있습니다.

ls 또는 dir : 현재 폴더의 파일/폴더 목록을 확인합니다.

cd [폴더명] : 해당 폴더로 이동합니다. (cd ..은 상위 폴더 이동)

mkdir [폴더명] : 새로운 폴더를 만듭니다. (예: mkdir 2026-04-24)

cp -Recurse [원본] [대상] : 폴더 전체를 복사합니다. (백업용 복사 시 유용)

mv [이름] [새이름] : 파일이나 폴더의 이름을 바꾸거나 이동시킵니다.

rm -Recurse [폴더명] : 폴더와 그 안의 내용물을 모두 삭제합니다.



3. 시스템 및 네트워크 확인 (톰캣 서버 운영 시)
서버 환경을 체크하거나 포트 충돌을 확인할 때 자주 쓰입니다.

Get-Process : 현재 실행 중인 프로세스 목록을 보여줍니다. (톰캣이 떠 있는지 확인)

netstat -ano | findstr :8080 : 8080 포트를 사용 중인 프로세스를 찾습니다. (포트 중복 확인)

ipconfig : 현재 내 PC의 IP 주소를 확인합니다.

cat [파일명] : 텍스트 파일(로그 파일 등)의 내용을 터미널에서 바로 읽습니다.



4. Git 관련 명령어 (GitHub 백업 시)
GitHub에 비공개로 올리실 때 이 3단계만 기억하시면 됩니다.

git add . : 현재 폴더의 모든 변경 사항을 스테이징 영역에 올립니다.

git commit -m "날짜별 백업 메시지" : 변경 사항을 확정합니다.

git push origin main : GitHub 서버로 업로드합니다.
